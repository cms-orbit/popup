# Changelog

이 문서는 `cms-orbit/popup`의 릴리스 노트를 기록합니다.

## 4.0.1 - 2026-07-05

### 변경

- 팝업 본문 편집을 `RichText`(Notion형 블록 에디터)로 전환했습니다.
- `cms-orbit/core` `4.0.1` 필수 (Quill 필드 제거 breaking change).

## 4.0.0 - 2026-07-05

### 변경

- 4.0.0 정식 릴리스로 승격했습니다. `popups` 테이블은 이미 단일 create 마이그레이션으로 유지됩니다.

### 추가

- Laravel Boost 가이드라인·스킬(`popup-development`)과 Vite alias manifest(`resources/orbit/frontend.json`)를 추가했습니다.

### 개선

- README **호스트 설정** 표를 추가해 필수 작업(`migrate`, `orbit:frontend-sync`, `<OrbitPopups />` 1줄)과 불필요한 수동 라우트 편집을 구분했습니다.
- README에 Laravel Boost 연동과 설치 버전(`^4.0`)을 정리했습니다.
- `cms-orbit/core` `4.0.0` 정식 릴리스에 맞춰 의존성을 정렬했습니다.
- 패키지 한글팩이 PHP `__()` 경로에서도 안정적으로 로드되도록 번역 경로 등록 시점을 `register()`로 앞당겼습니다.

### 수정

- 코어 부트 단계에서 번역 캐시가 먼저 고정되면서 팝업 관리자 문구가 영어로 남을 수 있던 문제를 수정했습니다.

## 4.0.0-beta4 - 2026-07-04

### 개선

- 이제 더 이상 팝업 하단 액션 버튼이 코드에 고정된 한국어 문구로만 남지 않고, 한글팩 키를 통해 일관되게 관리되도록 정리했습니다.
- `cms-orbit/core` `4.0.0-beta4` 기준으로 버전선을 맞춰 팝업 패키지와 코어 패키지 조합을 더 명확하게 유지할 수 있게 했습니다.

### 수정

- `OrbitPopups` 오버레이의 “오늘 하루 보지 않기”, 닫기 버튼이 번역팩을 통하지 않던 문제를 수정했습니다.
- 패치노트에 실제 프런트 번역 보강 내용이 빠져 있던 부분을 보완했습니다.

## 4.0.0-beta3 - 2026-07-04

### 안내

- 설치, 활성 팝업 API, `OrbitPopups` 컴포넌트 연결 흐름을 README에 정리해 바로 붙여 쓸 수 있도록 문서를 보강했습니다.

### 개선

- 이제 더 이상 팝업 표시 컴포넌트를 어디에 연결해야 할지 소스 코드를 뒤지지 않아도, 레이아웃 예시와 엔드포인트 설명을 README에서 바로 확인할 수 있습니다.
- `cms-orbit/core`와의 의존성 버전을 `4.0.0-beta3` 릴리스 선으로 정리해, Orbit 패키지 조합이 더 명확해졌습니다.

### 수정

- "오늘 하루 보지 않기"와 세션 단위 닫기 동작이 어떤 기준으로 저장되는지 문서가 비어 있어 운영자가 동작을 추측해야 했던 부분을 보완했습니다.
- 릴리스 기록이 없어 어떤 버전에서 활성 팝업 오버레이를 기준 삼아야 하는지 파악하기 어려웠던 문제를 정리했습니다.

## 4.0.0-beta2 - 2026-07-04

### 추가

- Orbit 문서 엔진 기반 `PopupEntity`와 공개 `popups.active` 엔드포인트를 추가했습니다.
- 노출 시작/종료 시각, 무시 일수, 스타일 JSON, 타이틀 표시 여부를 관리할 수 있는 팝업 전용 필드를 제공했습니다.
- 프런트에서 현재 활성 팝업을 가져와 바로 띄울 수 있는 `OrbitPopups` 오버레이 컴포넌트를 추가했습니다.

### 개선

- 이제 더 이상 팝업 공개 API와 관리자 엔티티를 따로 이어붙이지 않아도, 패키지 설치만으로 CRUD와 공개 조회 흐름이 함께 연결되도록 개선되었습니다.

### 수정

- 문서 섹션 등록과 번역 경로 연결이 분산되어 확장 패키지로 쓰기 번거롭던 부분을 정리했습니다.
